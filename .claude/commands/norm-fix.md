description: "指定ファイルまたは変更ファイルの 42 Norm 準拠を確認し、違反箇所を修正する"
---

以下の手順で Norm チェックと自動修正を行ってください。

## 1. 対象ファイルの特定

引数でファイルが指定されていればそのファイルを、なければ以下で変更ファイルを取得:

```bash
git diff --name-only HEAD -- '*.c' '*.h'
git diff --cached --name-only -- '*.c' '*.h'
```

## 2. 各ファイルについて以下をチェック・修正

### 関数の行数（25 行以内）
- 中括弧 `{` `}` のみの行はカウントしない
- 25 行を超える関数はヘルパー関数に分割する

### 関数の数（1 ファイル 5 関数以内）
- `static` 関数も含めてカウント
- 超過する場合は別ファイルに分離

### 行の長さ（80 文字以内）
- タブは 4 文字としてカウント
- 長い行は改行して整形

### 変数宣言
- 関数内の変数宣言は全て先頭にまとめる
- 途中の宣言は先頭に移動

### for ループ
- `for` を見つけたら等価な `while` に書き換え

### 引数の数（最大 4 つ）
- 超過する場合は構造体にまとめることを提案

## 3. 修正を適用

修正が必要なファイルは直接編集してください。修正内容をまとめて報告。
