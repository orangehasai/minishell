# Minishell

42 の課題。Bash の基本機能を再現したシェルを C で実装する。2名チーム。

## ビルド・テスト

```bash
make                # ビルド
make re             # クリーンビルド
make clean          # .o 削除
make fclean         # .o + 実行ファイル削除
./minishell         # 実行
valgrind --leak-check=full --suppressions=readline.supp ./minishell  # リーク確認
norminette src/ includes/  # Norm チェック
```

## 42 Norm v4.1（必須）

違反すると評価 0 点。完全な仕様は `docs/norm_v4.1.pdf` を参照。
norminette が検出しないルール（*印）もレビュワーが発見すれば不合格になり得る。

### 命名規則
- 構造体名: `s_`、typedef名: `t_`、共用体名: `u_`、列挙型名: `e_`、グローバル変数名: `g_`
- すべての識別子はスネークケース（小文字+数字+`_`のみ）。大文字禁止
- ファイル名・ディレクトリ名もスネークケース
- (*) 識別子は英語として読める明示的な名前にする
- `const`/`static` なしのグローバル変数は禁止（プロジェクトで許可された場合を除く）

### フォーマット
- 1関数 25行以内（波括弧 `{` `}` の行は除く）
- 1行 80文字以内（タブは4文字分としてカウント）
- インデントは半角スペースではなくタブ（4文字幅）
- 波括弧はそれ単独の行に配置（struct/enum/union の宣言部分は除く）
- 空行は文字を含まない（空白やタブも不可）。行末の空白/タブも不可
- 空行は連続禁止。空白記号も連続禁止
- 変数宣言は関数の冒頭のみ。宣言と初期化を同一行で行うのは禁止（static/const を除く）
- 1行に1変数宣言。変数名はスコープ内で同じ列にインデント
- ポインタの `*` は変数名に隣接させる（`char *str` ○、`char * str` ✗）
- 変数宣言とコード本体の間に空行1行。関数内に他の空行は不可
- 1行に1命令/制御構造のみ。制御構造内での代入は禁止
- カンマ/セミコロンの後にスペース。各演算子/被演算子はスペースで区切る
- Cキーワードの後にスペース（型名 int/char/float 等と sizeof は除く）
- 制御構造内は波括弧で囲む（単一行に単一命令の場合は除く）

### 関数
- 引数は最大4つの名前付きパラメータ。引数なしの場合は `void` を明示
- プロトタイプの引数にも名前が必要
- 関数内の変数宣言は最大5つ
- `return` の戻り値は丸括弧で囲む: `return (val);`（戻り値なしの場合は `return ;`）
- プロトタイプでは戻り値型と関数名の間にタブ1つ

### 禁止事項
- `for`、`do...while`、`switch`、`case`、`goto`
- 三項演算子 `?:`
- 可変長配列 (VLA)
- 変数宣言の暗黙的な型

### ファイル
- 1 .c ファイルに関数定義は最大5つ
- .c ファイルで .c ファイルをインクルード禁止
- 構造体の宣言は .h ファイルのみ（.c ファイル内での struct 宣言禁止）
- ヘッダファイルには二重インクルードガード必須（`FT_FOO_H` 形式）
- (*) 未使用のヘッダをインクルード禁止
- すべての .c/.h ファイルの先頭に 42ヘッダーを配置

### コメント
- 関数内にコメントを配置してはならない（行末コメントも不可）
- コメントは関数間に、行末またはコメント単独の行として配置
- (*) コメントは英語で記述し有用であること

### マクロ
- `#define` はリテラルまたは定数値のためのみ使用可
- 複数行にわたるマクロは禁止。マクロ名はすべて大文字
- グローバルスコープ外でのプリプロセッサ命令は禁止

### Makefile
- `$(NAME)`、`all`、`clean`、`fclean`、`re` ルール必須
- 不要な再リンク禁止
- ソースファイル名を明示列挙（`*.c` 等のワイルドカード禁止）
- libft は Makefile から自動コンパイル

## コーディング規約

- `malloc` の戻り値は必ず NULL チェック
- heap 確保したメモリは使用後に必ず `free`
- `ft_strjoin` 等の中間文字列は即座に free（リークの最頻出原因）
- エラーメッセージは `minishell: ` プレフィックス付きで stderr に出力
- errno 系エラーは `perror` を使う
- 関数が長くなったらヘルパー関数に分割（Norm 25 行制限）

構造体定義の詳細は `includes/minishell.h` を参照。

## よくあるミス（Gotchas）

- `he"ll"o` は 1 つの word。クォートの切れ目で word を分割してはいけない。
- `$` の後に変数名がない場合（`$` 単体、`$""`）はリテラルの `$` として残す。
- `>` のリダイレクトは左から右に適用。`> a > b` は a を空で作成し b に書き込む。
- builtin はパイプ外では fork しない（cd, export が親プロセスの状態を変えるため）。
- builtin をパイプ内で実行する場合は fork が必要。
- heredoc のデリミタにクォートがある場合、内容の変数展開を無効にする。
- `readline()` 由来のリークは修正不要（subject に明記）。自前コードのリークのみ対応。
- `PATH` が unset された場合、PATH 検索を行わない（絶対/相対パスのみ受け付ける）。
- `exit` の引数が 2 つ以上の場合、エラーを出すが終了はしない。
- シグナルハンドラ内では async-signal-safe な関数のみ使用可（write は OK、printf は NG）。


## 最初の指示
docs/配下を読み、プロジェクト概要を理解する。
実装前に不明点がある場合は逐一確認して。
実行テスト後は"make clean", "make fclean"を実行してクリーンナップしてください。
tests/はnormの制約を受けません。norminetteを実行する際は /src /includesを引数に渡してください。
